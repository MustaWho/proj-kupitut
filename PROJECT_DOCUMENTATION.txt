Документация проекта КупиТут

1. Общее описание проекта

КупиТут - учебный интернет-магазин с фронтендом на React и бэкендом на FastAPI.
Проект позволяет пользователям просматривать товары, добавлять их в корзину,
оформлять заказ, оставлять отзывы, получать рекомендации и работать с личным
кабинетом.

В проекте реализованы три роли:

- Покупатель - просматривает товары, добавляет товары в корзину, оформляет
  заказ, оставляет отзывы и загружает аватар.
- Продавец - управляет только своими товарами: добавляет, редактирует,
  удаляет товары, назначает скидку и загружает изображение товара.
- Администратор - управляет всеми товарами, категориями, пользователями,
  блокировками аккаунтов и просматривает историю продаж.

3. Backend: папка app

app/main.py

Главная точка входа FastAPI-приложения.
Здесь создается приложение, подключаются маршруты API, CORS, обработчики ошибок,
ограничение количества запросов и раздача файлов из папки /media.

app/config.py

Файл настроек приложения.
Здесь задаются:

- название API;
- версия приложения;
- адрес базы данных;
- настройки CORS;
- настройки токенов;
- папка для пользовательских изображений media.

app/models.py

Описание моделей базы данных через SQLAlchemy.
Основные модели:

- User - пользователь сайта.
- Category - категория товара.
- Product - товар.
- Review - отзыв к товару.
- CartItem - товар в корзине.
- ProductView - история просмотров товара.
- Promotion - рекламная акция.
- Order - заказ.
- OrderItem - товарная строка заказа.
- RefreshToken - refresh-токен пользователя.

Модель User хранит email, логин, имя, фамилию, роль, статус активности и ссылку
на аватар.

Модель Product хранит название, описание, категорию, цену, рейтинг, скидку,
изображение, статус активности и продавца товара.

Модель Review хранит оценку, текст отзыва, связь с пользователем и товаром.

Модель Order и OrderItem используются для истории заказов и продаж.

app/controllers/

Папка с API-маршрутами.

app/controllers/router.py

Собирает все маршруты API в единый роутер /api/v1.

app/controllers/auth.py

Маршруты регистрации, входа, обновления токена и смены пароля.

Основные endpoint:

- POST /api/v1/auth/register
- POST /api/v1/auth/login
- POST /api/v1/auth/refresh
- POST /api/v1/auth/change-password

app/controllers/users.py

Маршруты работы с пользователями.

Основные endpoint:

- POST /api/v1/users - создание пользователя.
- GET /api/v1/users - список пользователей.
- GET /api/v1/users/{user_id} - получение пользователя по ID.
- PATCH /api/v1/users/{user_id} - обновление пользователя.
- DELETE /api/v1/users/{user_id} - удаление пользователя.
- POST /api/v1/users/{user_id}/avatar - загрузка или замена аватара.

app/controllers/shop.py

Основные маршруты интернет-магазина.

Здесь реализованы:

- категории;
- товары;
- отзывы;
- история просмотров;
- рекомендации;
- корзина;
- оформление заказа;
- рекламные акции;
- история продаж.

Ключевые endpoint:

- GET /api/v1/categories
- POST /api/v1/categories
- GET /api/v1/products
- POST /api/v1/products
- GET /api/v1/products/{product_id}
- PATCH /api/v1/products/{product_id}
- DELETE /api/v1/products/{product_id}
- POST /api/v1/products/{product_id}/image
- POST /api/v1/products/{product_id}/reviews
- POST /api/v1/users/{user_id}/views
- GET /api/v1/users/{user_id}/recommendations
- POST /api/v1/users/{user_id}/cart
- GET /api/v1/users/{user_id}/cart
- PATCH /api/v1/users/{user_id}/cart/{item_id}
- DELETE /api/v1/users/{user_id}/cart/{item_id}
- POST /api/v1/users/{user_id}/checkout
- GET /api/v1/promotions
- POST /api/v1/promotions
- GET /api/v1/sales

app/controllers/dependencies.py

Содержит зависимость get_db для получения сессии базы данных.

app/dto/

Папка со схемами данных Pydantic.
Эти схемы описывают, какие данные API принимает и возвращает.

app/dto/user.py

Схемы пользователей:

- UserCreate - создание пользователя.
- UserUpdate - обновление пользователя.

app/dto/auth.py

Схемы авторизации:

- LoginRequest;
- TokenPair;
- AuthResponse;
- RefreshRequest;
- ChangePasswordRequest.

app/dto/shop.py

Схемы интернет-магазина:

- ReviewCreate и ReviewRead;
- CartItemCreate, CartItemUpdate и CartRead;
- CheckoutCreate;
- OrderRead;

app/repositories/

Слой работы с базой данных.
Репозитории отвечают за запросы к таблицам и сохранение изменений.

app/repositories/user.py

CRUD-операции для пользователей:

- получить пользователя по ID;
- найти по email;

- получить список;
- создать;
- обновить;
- удалить.

app/repositories/shop.py

Операции магазина:

- работа с категориями;
- работа с товарами;

- работа с корзиной;
- работа с рекламными акциями;
- получение рекомендаций;

app/services/

Слой бизнес-логики.
Сервисы проверяют правила приложения перед обращением к репозиториям.

Регистрация, вход, проверка пароля, создание access-token и refresh-token.

app/services/user_service.py

Логика работы с пользователями:

Главная бизнес-логика магазина:

Вспомогательные функции.

9. Ключевой реализованный функционал

Каталог товаров

Пользователь может открыть каталог, искать товары, фильтровать по категории,
цене и рейтингу. Каждый товар отображается карточкой с изображением, ценой,
скидкой и рейтингом.

Карточка товара

Показывает полное описание товара, изображение, цену, скидку, рейтинг и отзывы.
Покупатель может оставить оценку и отзыв.

Корзина

Корзина хранится в localStorage. Пользователь может добавлять товары, менять
количество и оформлять заказ.

Оформление заказа

После оформления заказа создается запись Order и строки OrderItem. Эти данные
используются в истории продаж администратора.

Отзывы и рейтинг

Рейтинг товара не вводится вручную при создании товара.
Он рассчитывается автоматически на основе оценок из отзывов покупателей.

Рекомендации

Когда пользователь открывает карточку товара, просмотр сохраняется.
На основе истории просмотров система предлагает товары похожих категорий.

Роли пользователей

Покупатель работает с каталогом, корзиной, отзывами и кабинетом.
Продавец управляет своими товарами.
Администратор управляет товарами, категориями, пользователями и продажами.

Загрузка изображений

Пользователь может загрузить аватар.
Продавец или администратор может загрузить изображение товара.
Файлы сохраняются локально в media и заменяются по ID, чтобы не переполнять
папки старыми изображениями.

Панель администратора

Администратор может:

- искать пользователей;
- открывать профили пользователей;
- менять роль пользователя;
- блокировать и разблокировать аккаунты;
- смотреть список заблокированных;
- смотреть историю продаж;
- искать товары по ID;
- редактировать и удалять любые товары.

12. Что важно помнить

Тестовые данные

При запуске через Docker тестовые пользователи, категории и товары создаются автоматически после миграций.
Каталог не остается пустым после команды docker compose up --build.

Для локального запуска без Docker используется скрипт:

python -m app.seed_demo

Скрипт создает пользователей:

- admin@kupitut-shop.ru, логин admin, пароль secret999, роль администратор;
- buyer@kupitut-shop.ru, логин buyer, пароль secret999, роль покупатель;
- seller@kupitut-shop.ru, логин seller, пароль secret999, роль продавец.

Также создаются категории и несколько тестовых товаров от имени продавца.
Если запустить скрипт повторно, дубли пользователей, категорий и товаров не
создаются.

После изменений базы данных нужно применять миграции:

.\.venv\Scripts\python.exe -m alembic upgrade head

Если изображения не отображаются, нужно проверить:

- запущен ли backend;
- есть ли файлы в media/avatars или media/products;
- проксируется ли /media через Vite или Nginx.

Если вход не работает, нужно проверить:

- зарегистрирован ли пользователь;
- активен ли аккаунт;
- не заблокирован ли пользователь администратором;
- правильный ли пароль.
